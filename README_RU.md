# Admissions Evaluation Engine

AI-система оценки кандидатов на основе эссе и коротких ответов. Она преобразует неструктурированные ответы кандидата в структурированную, объяснимую оценку с баллами, обоснованием, итоговым выводом, уровнем уверенности и флагами для проверки.

Система предназначена для использования в формате **human-in-the-loop**. Она не принимает решения самостоятельно, а помогает комиссии принимать более обоснованные и прозрачные решения.

---

## Обзор проекта

Приёмным комиссиям часто приходится обрабатывать большое количество заявок, сохраняя при этом:
- объективность
- последовательность оценки
- прозрачность решений

Данный проект решает эту задачу за счёт комбинации:

- структурированных промптов для стабильного ответа модели  
- валидации схемы для предсказуемого результата  
- детерминированного расчёта итогового балла  
- явных флагов для ручной проверки  

В результате получается **простая и объяснимая AI-система оценки**, которую легко понимать, тестировать и расширять.

---

## Почему это важно

Система помогает:

- снизить разброс оценок между разными проверяющими  
- быстрее находить сильных кандидатов  
- выявлять слабые или шаблонные ответы  
- повышать прозрачность процесса отбора  
- поддерживать человека, а не заменять его  

---

## Основные возможности

- Оценка кандидатов по 6 критериям:
  - мотивация
  - лидерство
  - потенциал роста
  - инициативность
  - коммуникация
  - аутентичность
- Формирование объяснения по каждому критерию
- Возврат:
  - сильных сторон
  - рисков (concerns)
  - итогового summary
  - confidence (уверенности)
- Детерминированный расчёт `overallScore`
- Формирование флагов:
  - highPotential
  - needsReview
  - lowData
- Валидация всех ответов модели через Zod
- Чистое разделение логики по слоям

## Архитектура

Проект построен модульно:

```
src/
  prompts/        → формирование промптов
  services/       → работа с OpenAI
  parsing/        → парсинг и валидация
  domain/         → логика скоринга и флагов
  schemas/        → типы и схемы (Zod + TS)
  evaluation/     → оркестрация
  fixtures/       → тестовые кандидаты
```

---

## Как это работает

1. Валидируется входной кандидат  
2. Формируется структурированный prompt  
3. Модель генерирует JSON-ответ  
4. Ответ парсится и валидируется  
5. Считается итоговый балл (`overallScore`)  
6. Вычисляются флаги  
7. Возвращается финальный результат  

---

## Сравнение с базовым подходом

Чтобы показать эффективность, система сравнивается с простым baseline.

### Baseline:
- длина эссе  
- количество ответов  
- наличие ключевых слов ("team", "leader", и т.д.)

### Пример:

Strong → Baseline: 6.2 | AI: 8.12  
Average → Baseline: 5.9 | AI: 5.88  
Weak → Baseline: 5.8 | AI: 3.94  

AI даёт **более точную и адекватную оценку**, особенно для слабых кандидатов, где baseline переоценивает качество.

## Демонстрация

В демо используются три типа кандидатов:

- сильный кандидат  
- средний кандидат  
- слабый кандидат  

Это показывает, как система:
- различает уровень кандидатов  
- объясняет свои оценки

## Ограничения

- зависит от качества модели  
- не проверяет факты кандидата  
- confidence генерируется моделью  
- используется демо-данные  
- нет backend/API слоя  

---

## Этические аспекты

- система не принимает решения самостоятельно  
- не использует чувствительные признаки  
- избегает bias  
- предоставляет объяснение оценки  
- финальное решение остаётся за человеком
